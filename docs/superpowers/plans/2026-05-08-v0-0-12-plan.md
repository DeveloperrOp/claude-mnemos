## Task 1: KPI Lost = filtered count (Spec §B)

**Files:**

**Why:** Today `_compute_lost_total` counts every uningested transcript on disk including `agent-*` sub-agent fragments and sessions <24h old. - [ ] **Step 1: Extract `_scan_all_vaults` body into a reusable helper**

The current function lives inside `claude_mnemos/daemon/routes/lost_sessions.py:59` and is closure-coupled to a `Request`. Extract a pure helper that takes runtimes directly, so the dashboard can call it without faking a request:

Open `claude_mnemos/daemon/routes/lost_sessions.py` and add this function ABOVE `_scan_all_vaults`:

```python
def collect_lost_sessions(runtimes: list[Any]) -> list[dict[str, Any]]:
    """Pure helper: cross-vault scan with cwd-based attribution + dedupe.

    Same filtering as scan_lost_sessions (mtime ≥ 24h ago, no agent-* files,
    not in any vault's manifest). Reusable from dashboard.snapshot.
    """
    global_ingested = global_ingested_shas(runtimes)
    resolver = ProjectResolver()
    seen_shas: set[str] = set()
    out: list[dict[str, Any]] = []
    for runtime in runtimes:
        cache = runtime.lost_sessions_cache
        items = (
            cache.get_or_scan(runtime.vault_root)
            if cache is not None
            else core_lost_sessions.scan_lost_sessions(runtime.vault_root)
        )
        for item in items:
            if item.sha in seen_shas:
                continue
            if item.sha in global_ingested:
                continue
            seen_shas.add(item.sha)
            assigned = UNASSIGNED_PROJECT
            if item.cwd:
                try:
                    entry = resolver.resolve_by_cwd(Path(item.cwd))
                    if entry is not None:
                        assigned = entry.name
                except (ResolverAmbiguityError, OSError):
                    pass
            d = item.model_dump(mode="json")
            d["project_name"] = assigned
            out.append(d)
    return out
```

Then change `_scan_all_vaults` to a thin shim:

```python
def _scan_all_vaults(request: Request) -> list[dict[str, Any]]:
    return collect_lost_sessions(list(all_runtimes(request)))
```

```python
class StoredAlert(BaseModel):
    model_config = ConfigDict(extra="forbid")

    id: str
    detector: str
    severity: Severity
    message: str  # English fallback; kept for v0.0.11- on-disk alerts

    # otherwise falls back to the literal `message`. Detectors may emit
    # both — the frontend prefers i18n_key.
    i18n_key: str | None = None
    i18n_params: dict[str, Any] = Field(default_factory=dict)
    context: dict[str, Any] = Field(default_factory=dict)
    first_seen: datetime
    last_seen: datetime
    silenced_until: datetime | None = None
    dismissed: bool = False
```

```python
def _make(
    *,
    id_: str,
    detector: str,
    severity: str,
    message: str,
    i18n_key: str,
    i18n_params: dict[str, Any] | None = None,
    context: dict[str, Any] | None = None,
    now: datetime | None = None,
) -> StoredAlert:
    n = now if now is not None else utcnow()
    return StoredAlert(
        id=id_,
        detector=detector,
        severity=severity,  # type: ignore[arg-type]
        message=message,
        i18n_key=i18n_key,
        i18n_params=i18n_params or {},
        context=context or {},
        first_seen=n,
        last_seen=n,
    )
```

`i18n_key` is REQUIRED (positional-keyword) so a missing one is a type error at call time, not a silent-default English-only alert.
